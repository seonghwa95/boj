### 문제 설명
어떤 수의 약수의 개수와 약수 전체를 입력받고
어떤 수의 약수 집합인지 구하는 문제

### 소스 코드
+ 메모리 : 30840 KB
+ 시간 : 68 ms

```python
n = int(input())
li = list(map(int, input().split()))
li.sort()

a = li[0]
b = li[-1]

print(a*b)
```
### 사고과정
각 수마다 약수의 개수가 다름을 고려하여
각각의 변수에 입력을 받기는 불가능 >> list로 입력받기

약수의 특징 고려하기
가장 작은 약수 * 가장 큰 약수 >> 구하고자 하는 수
두번째로 작은 약수 * 두번째로 큰 약수 >> 구하고자 하는 수
...
임을 고려하여 리스트를 오름차순으로 정렬한 후
리스트의 인덱스를 활용하여 가장 작은 수와 가장 큰 수를 구하였다.

### 코드설명
코드 자체는 매우 쉬우므로 생략
sort의 디테일한 문법이 생각나지 않아 구글링함..
+ 시간 복잡도 $$O(n^2)$$
으로 추측..?
다른 과정은 Big-O 상수 시간에 해결이 되고
sort 과정이 이중 루프를 통해 각 수를 비교해 나가는 식으로 진행이 될 것이라고 판단했다.
